# 短视频拉片与复刻 Skill

把一条对标视频拆成可学习、可执行、可复用的创作说明书。

这个仓库面向短视频创作者、运营和内容团队：输入视频文件、链接、截图或作品名称，输出一份带关键帧的 HTML 拉片报告，并补一份复刻 SOP。重点不是写影评，而是看清一条视频如何组织信息、如何制造信任、如何靠画面和剪辑推进观看。

## 适合做什么

- 拆解小红书、视频号、抖音、YouTube 等平台的对标视频
- 还原爆款视频的开头、节奏、段落、字幕、图形层和收口方式
- 为自己的选题生成可执行的拍摄、剪辑和包装步骤
- 对影视、剧集、纪录片、教程、评测等内容做结构化视听分析

## 默认产物

1. **拉片报告**：结论、关键发现、逐段配帧表、视觉系统、节奏判断。
2. **复刻 SOP**：从选题、素材、口播、拍摄到剪辑包装的执行步骤。
3. **单文件 HTML**：关键帧内嵌，离线可读，便于发给团队讨论。

## 工作流

```text
定目标 -> 拆素材 -> 判类型 -> 看细节 -> 填逐段表 -> 产出报告和 SOP
```

先确定这条视频要学习什么，再抽帧和分析。短视频尤其要分清“内容层”和“制作层”：

- 内容层：选题、钩子、段落推进、逐字稿、信息密度、CTA
- 制作层：镜头、构图、字幕、字体、图形元素、转场、音效、配乐、工具链

## 输入方式

| 输入 | 用法 |
|---|---|
| 作品名称 | 适合讨论已有作品，基于共同认知做结构化分析 |
| 截图 / 字幕 | 适合快速看版式、台词、关键画面 |
| 本地视频 / URL | 适合完整拉片：切场景、抽关键帧、建时间轴 |

## 短视频报告结构

1. 视频概况：标题、时长、切点数量、内容类型
2. 一句话判断：这条视频靠什么成立
3. 拉片维度：本次重点看哪些列
4. 可复刻度：内容、素材、拍摄、图形、节奏、工具链
5. 关键发现：3-5 条可直接迁移的规律
6. 逐段配帧表：时间码、代表帧、台词/字幕、记忆点、图形层、作用
7. 视觉清单：字体、颜色、字幕、贴纸、标注线、画中画、转场
8. 确定性标注：事实 / 高置信推断 / 低置信推断 / 未知
9. 复刻步骤：按用户目标改写成可执行动作

```bash
npx skills add Rinkon2046/lapian.skill
```

## 系统依赖

- `ffmpeg`：切场景、抽关键帧
- `yt-dlp`：从 URL 下载视频，可选
- `whisper`：无字幕时转写音频，可选
